## 1. キャラクター設計思想

### 1.1 原作の特徴を活かす

ガールズ＆パンツァーの魅力は、個性豊かなキャラクターたちのチームワークにある。
本システムでは、各キャラクターの原作における特徴を最大限に活かし、
マルチエージェントシステムの各役割に適切にマッピングしている。

#### 設計原則

| 原則 | 説明 |
|------|------|
| **個性の尊重** | 各キャラクターの口調・性格を忠実に再現 |
| **役割適合** | キャラクターの特性に合った役割を割り当て |
| **チームワーク** | 原作での関係性を活かした連携構造 |
| **学校カラー** | 各学校の特色を中隊の特性として反映 |

### 1.2 口調・性格の設定方針

#### 口調設定

各キャラクターには `speech_patterns` として定型フレーズを設定している。
これにより、エージェントの出力が原作の雰囲気を維持できる。

```yaml
speech_patterns:
  greetings:      # 挨拶
  acknowledgment: # 了解・確認
  proposals:      # 提案時
  completion:     # 完了報告
```

#### 性格設定

`personality.traits` でキャラクターの性格特性を定義し、
意思決定や応答スタイルに反映させている。

## 6. キャラクターのカスタマイズ方法

### 6.1 YAMLファイルの構造

各キャラクターは `characters/*.yaml` に定義されている。

```yaml
# 基本情報
character:
  id: character_id        # 一意のID
  name: キャラクター名
  name_en: EnglishName

# 役割
role:
  type: role_type         # platoon_leader, frontend, backend, test
  platoon: 1              # 所属中隊番号

# 性格
personality:
  traits:
    - 特性1
    - 特性2
  communication_style: スタイル説明
  decision_making: 意思決定スタイル

# 口調
speech_patterns:
  greetings:
    - "挨拶1"
  acknowledgment:
    - "了解フレーズ"
  completion:
    - "完了報告"

# エージェント設定
agent_config:
  model: claude-sonnet-4-20250514
  temperature: 0.7
  max_tokens: 4096
```

### 6.2 カスタマイズ可能な項目

| 項目 | 説明 | 影響 |
|------|------|------|
| `speech_patterns` | 口調パターン | 応答の雰囲気 |
| `personality.traits` | 性格特性 | 意思決定スタイル |
| `agent_config.temperature` | 応答のランダム性 | 創造性 vs 一貫性 |
| `responsibilities` | 担当業務 | タスク割り当て |

### 6.3 新規キャラクター追加手順

1. `characters/` に新しいYAMLファイルを作成
2. 必須フィールド（id, name, role）を記入
3. personality と speech_patterns を設定
4. `scripts/start_panzer.sh` にペイン追加

### 6.4 口調カスタマイズ例

```yaml
# より厳格な口調にする場合
speech_patterns:
  greetings:
    - "報告せよ"
  acknowledgment:
    - "承知した"
  completion:
    - "任務完了"
```
